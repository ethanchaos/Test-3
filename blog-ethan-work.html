Ethan Chaos: Projects & Work

A complete overview of the platforms, tools, and ongoing work built by Ethan Chaos — from the flagship Chaos Configurations site to the Chaos SNI Finder tool and beyond.

📖 ~5,500 words ⏱️ 22 min read 🏷️ Projects · Development

Project Overview

Ethan Chaos operates across two primary platforms — Chaos Configurations and VPN Config Hub — plus a growing suite of tools designed to support the practical free-net workflow. Each project serves a distinct purpose in the broader ecosystem he's building, and together they cover the full spectrum from educational content to practical tooling to hands-on infrastructure.

What distinguishes Ethan's portfolio from similar projects in the networking space is the combination of depth and accessibility. Most networking resources are either highly technical (written for engineers and assuming significant prior knowledge) or extremely simplified (written for non-technical users and sacrificing accuracy for accessibility). Ethan's work occupies the middle ground: technically rigorous enough to be useful for serious users, accessible enough to be approachable for motivated beginners.

Project 1: Chaos Configurations

Chaos Configurations is the flagship platform — the comprehensive knowledge and tooling hub that encompasses everything from beginner VPN guides to advanced protocol analysis. The site you're reading right now is Chaos Configurations.

What It Is

Chaos Configurations is a static web platform built for performance and accessibility. The design philosophy prioritizes speed on constrained connections, clean presentation of technical content, and a user experience that doesn't require JavaScript-heavy frameworks to function. It is built with plain HTML, CSS, and minimal JavaScript — intentionally lightweight so it loads fast on mobile connections with limited bandwidth, which is the primary access method for a significant portion of its audience.

Content Architecture

The site is organized around several content pillars:

  • Educational Guides: Deep-dive explanations of VPN technology, tunneling protocols, SNI mechanics, Freenet, and network restrictions — written to build genuine understanding, not just surface-level familiarity.
  • Config Hub: A library of tested, categorized VPN configuration files for HTTP Injector, HA Tunnel Plus, and OpenVPN — the practical resource for users who need working configs without necessarily building them from scratch.
  • Setup Guides: Step-by-step installation and configuration walkthroughs for the major free-net applications — accessible to beginners, complete enough to be useful for experienced users troubleshooting specific issues.
  • Research Articles: Longer-form analysis pieces covering the policy, history, and technical landscape of internet freedom technology — including pieces on zero-rating, digital access inequality, and the history of Freenet.
  • Tools: The Chaos SNI Finder and other networking utilities, presented with full documentation and usage guides.

Technical Stack

Chaos Configurations is built as a static site — no server-side rendering, no database, no complex backend. This choice is deliberate and reflects several priorities:

  • Performance: Static sites load faster than dynamic ones. For users on slow or expensive mobile connections, every kilobyte matters.
  • Reliability: Static sites have no backend to crash. The site is as reliable as the hosting infrastructure it runs on — and static hosting is extremely reliable.
  • Deployability: A static site can be deployed on Netlify, Cloudflare Pages, GitHub Pages, or virtually any web host — with zero configuration overhead and no server management required.
  • Security: No backend means no database injection attacks, no server-side vulnerabilities, no authentication systems to compromise.

The frontend is built with a design system using Google Fonts (Space Grotesk for body text, JetBrains Mono for code, Playfair Display for headings), a custom CSS framework, and a minimal JavaScript navigation system. The result is a site that looks professional and modern while remaining extremely lightweight.

Project 2: VPN Config Hub

VPN Config Hub (vpnconfighub.netlify.app) is a more focused, reference-oriented platform — designed as a clean, organized library of VPN resources rather than a full educational hub. Where Chaos Configurations spans the full spectrum from beginner education to advanced research, VPN Config Hub is designed for a user who already knows roughly what they need and wants to find it quickly.

What Makes It Different

VPN Config Hub's value proposition is clarity and organization. The site is structured as a navigable reference library: configs are categorized by protocol and app, guides are organized by task, and the navigation is designed to get users to what they need in as few clicks as possible.

It also serves as the external-facing deployment target for Ethan's Android app (ConfigHub — com.example.confighub), which loads the VPN Config Hub site in a WebView with native Android performance optimizations including hardware acceleration, WebView pre-initialization, Chrome Custom Tabs warmup, and skeleton loading screens.

The Android App Connection

The ConfigHub Android app is built around a WebView that loads the VPN Config Hub site, creating a native-app experience around the web content. The app is developed using AndroidIDE — an on-device Android development environment — and targets a wide range of Android devices (minSdk 21, compileSdk 34) to ensure maximum compatibility with older devices in developing markets where the latest hardware isn't universal.

Key app features include hardware-accelerated rendering for smooth scrolling, WebView pre-initialization for fast startup times, Chrome Custom Tabs warmup for improved link-opening performance, skeleton loading screens to improve perceived performance, and a custom error screen replacing Android's default WebView error page.

Project 3: Chaos SNI Finder

The Chaos SNI Finder is a specialized networking tool that automates the discovery and testing of SNI (Server Name Indication) hosts that work as effective bug hosts in carrier-bypass VPN configurations. It is Ethan Chaos's most technically distinctive contribution to the free-net ecosystem.

The Problem It Solves

Building a working HTTP Injector or HA Tunnel Plus configuration for a specific carrier requires knowing which SNI value to use as the "bug host" — the domain that the carrier's network recognizes as zero-rated or whitelisted and therefore allows traffic through without data charges. Finding this value manually is a time-consuming trial-and-error process: you have to test candidate domains one by one, observe the network response, and determine whether each one is genuinely zero-rated or just accessible because you have active data.

Chaos SNI Finder automates this process systematically. It tests candidate domains against the current network, distinguishes between "zero-rated" and "accessible via data," ranks working hosts by performance, and generates ready-to-use configuration parameters. What would take hours of manual testing is done in minutes.

How It Works

The tool operates through a multi-stage pipeline:

  1. Candidate generation: Builds a test list from known zero-rated domains, CDN infrastructure domains, government and educational domains, and previous discovery results.
  2. Zero-rating detection: Tests each candidate by attempting TLS connections with the candidate domain as the SNI value, comparing network behavior with and without active data balance to isolate genuinely zero-rated responses.
  3. Tunnel viability testing: Tests whether candidates that pass zero-rating detection can actually function as SNI values in SSH and SSL tunnel configurations — not all zero-rated domains are suitable as bug hosts due to carrier proxy validation behavior.
  4. Performance benchmarking: Measures latency and throughput for viable hosts to rank them by practical usability.
  5. Config output: Generates ready-to-use payload and SNI parameters for HTTP Injector (.ehi) and HA Tunnel Plus (.hat) configuration files.

Impact

The Chaos SNI Finder has meaningfully reduced the barrier to building and maintaining working free-net configurations. Before the tool, finding a working SNI host required either having inside knowledge of carrier networks or spending significant time on manual testing. With the tool, any user can run a systematic discovery process and get actionable results in minutes — regardless of their prior knowledge of how SNI works or how carrier networks are configured.

Infrastructure Work

Behind the visible platforms and tools, Ethan Chaos maintains a body of server infrastructure work — the VPS deployments, configuration management, and server optimization work that underpins the practical server-side recommendations on Chaos Configurations.

This work includes maintained OpenVPN and WireGuard server configurations optimized for performance in developing-market network conditions, SSH server configurations for use as backend servers for HTTP Injector and HA Tunnel Plus deployments, network performance tuning (TCP buffer sizing, BBR congestion control, MTU optimization), and security hardening configurations for VPS instances on DigitalOcean and Vultr.

These configurations form the basis of the server-side setup guides on Chaos Configurations — they're not theoretical recommendations but battle-tested configurations that have been deployed and validated against real network conditions.

Ongoing Development

Ethan Chaos continues to develop all three primary projects — Chaos Configurations, VPN Config Hub, and the Chaos SNI Finder — with updates driven by two sources: changes in the technical landscape (new carrier configurations, protocol updates, changes in filtering behavior) and community feedback (user-reported config failures, requests for coverage of new carriers or regions).

The development cadence reflects the nature of the domain: free-net technology is not a solved problem but a continuously evolving one. The carrier-bypass techniques that work today may be patched tomorrow. The SNI filtering systems that are cutting-edge today will be superseded by ECH-aware systems in the next few years. Staying current requires ongoing investment in research and tool maintenance — and that's the commitment Ethan has made.
