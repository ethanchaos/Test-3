Installing Burp Suite Community Edition

Get PortSwigger's industry-standard web proxy running on Kali Linux or via the JAR method on Termux with a desktop session. GUI required — two paths covered.

Your progress 0%
01

What is Burp Suite?

Burp Suite sits between your browser and the target web application — every request passes through it, giving you full visibility and control

Burp Suite Community Edition is PortSwigger's free web application security testing toolkit. It acts as an intercepting proxy between your browser and the target — letting you inspect, modify, and replay every HTTP/HTTPS request in real time. It's the go-to tool for web penetration testing and is used in virtually every bug bounty and web CTF challenge.

⚠ GUI required — Desktop environment mandatory
Burp Suite is a GUI-only application. There is no CLI-only mode. It requires a running desktop environment on every platform. On Android / NetHunter, start KEX (kex &), Termux-X11, or TigerVNC before launching. On a PC or laptop with a desktop Kali install, it works directly. Additionally, Burp Suite requires a 64-bit device — it will not run on 32-bit hardware or 32-bit Android phones.

Before you install, pick your platform:

✔
PC / Laptop (Kali Linux)
Easiest — apt install, one command, full GUI out of the box.
✔
Android — NetHunter / Kali (64-bit only)
Best mobile option — apt install inside Kali chroot, run via KEX or TigerVNC.
~
Android — Termux JAR method (64-bit only)
Hacky but works — install Java, download the JAR, run with a GUI session active.
✘
Termux only (no desktop)
Not possible — Burp Suite has no headless or CLI mode.
02

Installation

Two install paths — pick the one that matches your setup. Both need a desktop session running before you launch Burp.

On Kali and NetHunter (inside the Kali chroot), Burp Suite is in the official repos as burpsuite. One apt command installs everything.

bash 
$ sudo apt update
$ sudo apt install burpsuite -y
NetHunter users
Run this command from inside your Kali chroot session (after running nh or nethunter), not from the raw Termux shell. Then launch Burp from within your KEX or VNC desktop session.

Once installed, launch from within your desktop session:

bash 
$ burpsuite
Desktop session required first
Before running any of these steps, start KEX, Termux-X11, or TigerVNC. Without a display, the JAR will fail to open. Also requires a 64-bit Android device — 32-bit phones are not supported.

Step 1 — Install Java 17:

bash 
$ pkg update
$ pkg install openjdk-17

Step 2 — Download the Burp Suite JAR:

bash 
$ wget "https://portswigger.net/burp/releases/download\
?product=community&version=latest&type=Jar" \
  -O burp.jar

Step 3 — Run it (from inside your GUI session):

bash 
$ java -jar burp.jar
Allocate more RAM for better performance
Burp Suite can be memory-hungry. On Android where RAM is limited, add a heap flag: java -Xmx1g -jar burp.jar — this caps heap at 1 GB and prevents crashes on mid-range devices.
03

First Launch & Overview

When Burp Suite opens you'll see a startup wizard. For Community Edition, select Temporary project and click Next, then Start Burp. You'll land on the main dashboard.

The Burp Suite main interface — tabs across the top give you access to Proxy, Repeater, Intruder, Decoder and more from a single window

The key tabs you'll use immediately:

Burp Suite core tabs
TabWhat it does
ProxyIntercept and modify live HTTP/HTTPS requests from your browser
RepeaterManually re-send and tweak individual requests — essential for testing
IntruderFuzz parameters with wordlists (rate-limited in Community Edition)
DecoderEncode / decode Base64, URL, HTML, hex, and more
TargetSite map — a tree of every URL and endpoint Burp has seen
LoggerFull history of all proxied requests and responses

To start intercepting traffic, go to Proxy → Intercept and make sure Intercept is On. Then configure your browser to proxy through 127.0.0.1:8080 — Burp's default listener address.

HTTPS interception — install the CA cert
To inspect HTTPS traffic without errors, export Burp's CA certificate from Proxy → Options → CA Certificate and install it as a trusted authority in your browser. Without this, HTTPS sites will throw certificate warnings.
04

Troubleshooting

burpsuite: command not found
The package isn't installed. Run sudo apt update && sudo apt install burpsuite -y. If using Termux, make sure you're running the JAR method — Burp Suite is not available as a pkg package.
GUI won't open / cannot open display
No desktop is running. Start KEX (kex &), Termux-X11, or TigerVNC first, then run burpsuite or java -jar burp.jar from a terminal inside that session.
java -jar burp.jar — unsupported platform or architecture error
Your device is 32-bit. Burp Suite requires a 64-bit device — check with uname -m. If it shows armv7l or i686, Burp Suite cannot run on your hardware.
Burp crashes or runs out of memory on Android
Use the heap flag when launching: java -Xmx1g -jar burp.jar. Close other apps first and make sure you have at least 2 GB of free RAM. On very low-end devices Burp may be too heavy to run reliably.
Browser shows certificate error when intercepting HTTPS
You need to install Burp's CA certificate in your browser. Go to Proxy → Options → CA Certificate, export it, then add it to your browser's trusted certificate store. Each browser handles this differently — Firefox: Preferences → Certificates → Import.
Tutorial Complete!
Burp Suite is installed and running. Next: install the CA certificate in your browser and start intercepting your first HTTPS requests in the Proxy tab.
