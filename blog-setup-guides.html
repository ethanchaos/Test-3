Guide 01

HTTP Injector — Complete Setup Guide

HTTP Injector is an Android app that creates a VPN tunnel using SSH, SSL, or proxy protocols with custom HTTP header injection. It's the most powerful tool for carrier-bypass configurations because of its deep payload customization.

What You'll Need

  • Android device (Android 5.0 or higher)
  • HTTP Injector app (free on Google Play Store)
  • A config file (.ehi format) — get one from the Config Hub
  • Or: an SSH server and its connection details (if building a config from scratch)

Method 1: Importing a Config File

This is the fastest way to get connected. If you have a .ehi config file from the Config Hub:

  1. 1

    Download the config file

    Download the .ehi file from the Config Hub to your device's storage. Note where it's saved.

  2. 2

    Open HTTP Injector

    Launch the app. You'll see the main connection screen with a large connect button.

  3. 3

    Import the config

    Tap the menu icon (three lines, top left) → Import Config. Navigate to where you saved the .ehi file and select it.

  4. 4

    Review settings

    After import, review the loaded settings: SSH server, port, payload. You should not need to change anything if the config is current.

  5. 5

    Connect

    Tap the large power button. Accept the VPN permission request if prompted. The app will attempt to connect — watch the logs for success or error messages.

Method 2: Manual Configuration (SSH Mode)

For advanced users who want to build a config from scratch using their own SSH server:

  1. 1

    Open SSH Settings

    In the HTTP Injector main screen, tap SSH Settings. Enable SSH mode by toggling it on.

  2. 2

    Enter Server Details

    Enter your SSH server's IP address or hostname in the Server field. Enter the port (22, 80, or 443 depending on your server setup). Enter your SSH username and password.

  3. 3

    Configure the Payload

    The payload is the custom HTTP header that tells the carrier proxy where to route your connection. A basic CONNECT payload looks like:

    CONNECT [host_port] HTTP/1.1[crlf]Host: bug.carrier.com[crlf]X-Online-Host: bug.carrier.com[crlf][crlf]

    Replace bug.carrier.com with a working SNI/bug host for your carrier. Use the Chaos SNI Finder to discover one.

  4. 4

    Set Remote Proxy

    If your carrier routes through a proxy server, enter that proxy's IP and port in the Remote Proxy field. This is carrier-specific.

  5. 5

    Save and Test

    Save your configuration. Tap connect and monitor the log output. A successful connection will show "SSH tunnel established" and then assign a local IP. Test browsing to verify.

💡

Troubleshooting HTTP Injector

Connection timeout: The payload or proxy host isn't working. Try a different payload pattern or SNI host.
SSH auth failed: Wrong username/password or the server is blocking your IP. Try a different server.
Connected but no internet: DNS issue. Enable "DNS over tunnel" in settings.

Guide 02

HA Tunnel Plus — Complete Setup Guide

HA Tunnel Plus creates SSH tunnels with a simpler interface than HTTP Injector. It supports SSH over WebSocket, making it effective in environments where direct SSH is blocked but HTTPS WebSocket traffic is allowed.

What You'll Need

  • Android device
  • HA Tunnel Plus app (free — available via APK or official site)
  • A .hat config file — from the Config Hub
  1. 1

    Install HA Tunnel Plus

    Download and install the app. If installing via APK, enable "Install from unknown sources" in your Android security settings temporarily.

  2. 2

    Import Config File

    On the main screen, tap the + button or Import option. Select your .hat file. The config loads automatically with all settings pre-filled.

  3. 3

    Verify Connection Type

    Check that the connection type (SSH Direct, SSH over WebSocket, or SSH over SSL) matches what your config requires. This is shown in the config details screen.

  4. 4

    Connect

    Tap the connect toggle. Grant VPN permissions when prompted. HA Tunnel will show connection progress and then display connected status with a green indicator.

Guide 03

OpenVPN — Complete Setup Guide

OpenVPN is the most widely used open-source VPN protocol. It requires a server (either self-hosted on a VPS or provided by a service) and a .ovpn configuration file. It offers the best combination of security, compatibility, and flexibility.

What You'll Need

  • Android device
  • OpenVPN for Android app (open-source, on Google Play)
  • A .ovpn configuration file — from your server or the Config Hub
  • Optional: your own VPS — see the VPS Hosting guide
  1. 1

    Install OpenVPN for Android

    Install the app from Google Play. Use "OpenVPN for Android" by Arne Schwabe — the open-source version, not commercial alternatives.

  2. 2

    Import your .ovpn file

    Tap the folder icon in the top right. Select your .ovpn file. The app imports all settings including certificates embedded in the file.

  3. 3

    Connect

    Tap the imported profile name to connect. Grant VPN permissions. OpenVPN will show a detailed connection log — look for "Initialization Sequence Completed" to confirm success.

  4. 4

    Verify your IP

    Visit a site like whatismyip.com to confirm your IP address has changed to the VPN server's IP. If it shows your real IP, something is wrong with the routing.
