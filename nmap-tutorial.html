Installing Nmap — Complete Setup Guide

Install, verify, and run your first Nmap scan on Linux, macOS, and Windows. No prior experience needed.

Your progress 0%
01

What is Nmap?

Nmap maps network hosts and services across your infrastructure

Nmap (Network Mapper) is a free, open-source tool for network discovery and security auditing. It's used by network administrators to discover hosts and services on a network, and by security professionals to map attack surfaces.

What you'll be able to do
Discover live hosts, identify open ports, detect OS and software versions, and run scripted vulnerability scans — all from the command line.
02

Installing on Linux

Linux is the most common platform for Nmap. Most distros ship a package so installation is a single command.

bash 
$ sudo apt update
$ sudo apt install nmap -y
bash 
$ sudo dnf install nmap -y
bash 
$ sudo pacman -S nmap

Once installation finishes, verify it worked:

bash 
$ nmap --version
Nmap version 7.94 ( https://nmap.org )
Platform: x86_64-pc-linux-gnu
Compiled with: ...openssl...
03

Installing on macOS

The easiest way on macOS is Homebrew. If you don't have Homebrew yet, install it first.

Homebrew required
If you don't have Homebrew, visit brew.sh and paste the install command into your terminal first.
bash 
$ brew install nmap

Alternatively, Nmap provides a .dmg installer from the official website at nmap.org/download if you prefer a GUI installer.

04

Installing on Windows

Windows users get a GUI installer that also installs Zenmap (the graphical frontend) and Npcap (the packet capture driver Nmap depends on).

  1. Go to nmap.org/download.html and download the latest Windows self-installer (.exe)
  2. Run the installer as Administrator — accept the Npcap license when prompted
  3. Keep all default options checked (Npcap, register Nmap path)
  4. Click Install and wait for completion

After installation, open Command Prompt or PowerShell as Administrator and verify:

powershell 
PS> nmap --version
Nmap version 7.94 ( https://nmap.org )
Platform: i686-pc-windows-windows
05

Running Your First Scan

With Nmap installed, try a few foundational scans. Always test against hosts you own or have explicit permission to scan.

Legal reminder
Scanning networks you do not own or have written permission to test is illegal in most jurisdictions. Only scan your own lab or explicitly authorized targets.

Scan your local machine to test everything is working:

bash 
$ nmap localhost
Starting Nmap 7.94 ( https://nmap.org )
Nmap scan report for localhost (127.0.0.1)
Host is up (0.000052s latency).
Not shown: 997 closed tcp ports (conn-refused)
PORT     STATE SERVICE
22/tcp   open  ssh
80/tcp   open  http
5432/tcp open  postgresql

Nmap done: 1 IP address (1 host up) scanned in 0.04s
Common Nmap flags
FlagWhat it does
-sVDetect service versions on open ports
-OEnable OS detection (requires root)
-p 1-1000Scan ports 1 through 1000 only
-T4Aggressive timing (faster scan)
-AEnable OS + version detection + scripts
-oN out.txtSave output to a file
06

Troubleshooting

Hit a wall? Here are the most common issues and how to fix them.

nmap: command not found
The binary isn't on your PATH. On Linux run which nmap to check. On Windows, re-run the installer and make sure "Add Nmap to PATH" is checked.
OS detection requires root / Administrator
Use sudo nmap -O <target> on Linux/macOS. On Windows, open CMD or PowerShell as Administrator.
Scan shows all ports filtered
A firewall is likely blocking probes. Try nmap -Pn <target> to skip the ping check and scan anyway.
