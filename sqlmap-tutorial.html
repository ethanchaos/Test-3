Installing SQLmap — SQL Injection Tool Setup Guide

Get the world's most popular automated SQL injection framework running on Kali Linux or Termux. Pure CLI — works on any platform with Python 3.

Your progress 0%
01

What is SQLmap?

SQLmap automates the entire SQL injection workflow — from detection through database enumeration to credential extraction

SQLmap is an open-source penetration testing tool written in Python that automates the detection and exploitation of SQL injection vulnerabilities in web applications. Point it at a vulnerable URL parameter and it handles everything — fingerprinting the database engine, extracting tables, dumping credentials, and more.

Pure CLI — no GUI needed
Unlike Burp Suite or ZAP, SQLmap is entirely terminal-based. It runs natively on Kali, inside NetHunter, and directly in Termux on Android — no desktop environment, KEX, or VNC required.
Legal reminder
SQLmap itself displays this warning on every run — using it against targets without explicit permission is illegal. Only test on systems you own or have written authorisation to test. Use intentionally vulnerable practice apps like DVWA, WebGoat, or HackTheBox for learning.
02

Installation

SQLmap is a Python tool — it installs cleanly on both Kali and Termux. Pick your platform below.

SQLmap is pre-installed on most Kali builds. Check first — if it's already there, just update it. If not, one apt command installs it.

bash — Check if already installed 
$ sqlmap --version
1.10.4#stable

If that returns a version number, SQLmap is already installed. Update it with:

bash — Update existing install 
$ sudo apt update && sudo apt upgrade sqlmap -y

If SQLmap is not installed, install it fresh:

bash — Fresh install 
$ sudo apt update
$ sudo apt install sqlmap -y

On Termux, SQLmap isn't in the main pkg repo — install Python and git, then clone directly from the official GitHub repo. This gives you the latest version every time.

bash — Step 1: Update and install dependencies 
$ pkg update && pkg upgrade -y
$ pkg install python git -y
bash — Step 2: Clone SQLmap from GitHub 
$ git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git sqlmap-dev
bash — Step 3: Run it 
$ cd sqlmap-dev
$ python3 sqlmap.py --version
1.10.4#stable
Make a shortcut alias
To avoid typing python3 ~/sqlmap-dev/sqlmap.py every time, add an alias to your shell profile:

echo "alias sqlmap='python3 ~/sqlmap-dev/sqlmap.py'" >> ~/.bashrc && source ~/.bashrc

Now you can just type sqlmap like on Kali.

04

Basic Usage — SQL Injection Workflow

A typical SQLmap workflow follows these steps — always against targets you own or are authorised to test. The example URL used here is a local practice machine.

1
Test a URL for injection
Point SQLmap at a URL with a parameter and let it detect whether SQL injection is present.
2
Enumerate databases
Once injection is confirmed, list all databases on the server with --dbs.
3
List tables in a database
Target a specific database with -D dbname --tables to see what tables exist.
4
Dump table contents
Extract the data from a specific table with -D dbname -T tablename --dump.

Each of those steps as actual commands (replace the URL with your authorised target):

bash — Step 1: Test for injection 
$ sqlmap -u "http://target.local/page.php?id=1"
bash — Step 2: Enumerate databases 
$ sqlmap -u "http://target.local/page.php?id=1" --dbs
[*] available databases [2]:
[*] information_schema
[*] acuart
bash — Step 3: List tables 
$ sqlmap -u "http://target.local/page.php?id=1" -D acuart --tables
[8 tables]
+-----------+
| artists   |
| carts     |
| categ     |
| featured  |
| guestbook |
| pictures  |
| products  |
| users     |
+-----------+
bash — Step 4: Dump a table 
$ sqlmap -u "http://target.local/page.php?id=1" -D acuart -T users --dump
Essential SQLmap flags
FlagWhat it does
-u URLTarget URL with a parameter (required)
--dbsEnumerate all databases on the server
-D <db>Target a specific database
--tablesList tables in the specified database
-T <table>Target a specific table
--dumpExtract and display the table contents
--batchAuto-answer all prompts (non-interactive mode)
--random-agentUse a random User-Agent to reduce WAF detection
--proxy http://127.0.0.1:8080Route traffic through Burp Suite for analysis
-r file.txtLoad a saved HTTP request file (captured from Burp)
--level=5 --risk=3Increase test depth and payload aggressiveness
05

Troubleshooting

sqlmap: command not found on Kali
Run sudo apt install sqlmap -y. SQLmap is pre-installed on most Kali builds but may be missing from a minimal install. Check with apt-cache show sqlmap to confirm it's in your repos.
python3: command not found on Termux
Python isn't installed. Run pkg install python -y then retry. On Termux, python3 and python both point to the same binary after install.
SQLmap says the target is not injectable
The parameter may be protected or the test level is too low. Try adding --level=5 --risk=3 to run deeper tests. Also ensure the URL parameter you're targeting actually exists and returns different responses — test manually in a browser first.
WAF or firewall blocking requests
Add --random-agent to randomise the User-Agent header, or route through Burp with --proxy http://127.0.0.1:8080. For tougher WAFs, try tamper scripts: --tamper=space2comment,between.
SQLmap keeps prompting — I want it to run automatically
Add --batch to your command. This tells SQLmap to auto-accept all default answers to prompts and run without user interaction — useful for scripting or long sessions.
