Installing Metasploit Framework — CLI Setup Guide

Get the world's most widely used penetration testing framework running on Kali Linux or Termux (NetHunter). Two install paths, one goal — a working msfconsole.

Your progress 0%
01

What is Metasploit Framework?

Metasploit Framework — thousands of exploits, payloads, and auxiliary modules, all accessible from a single CLI console

The Metasploit Framework is an open-source penetration testing platform developed by Rapid7. It's the industry standard for exploit development, vulnerability research, and post-exploitation — used by security professionals, CTF players, and red teams worldwide.

Two install paths covered here
Kali / Linux — uses sudo apt install metasploit-framework. Full-featured, database-backed, updated via apt.

Termux (NetHunter) — uses pkg install metasploit via the unstable repo. Runs entirely on Android without root.
Legal reminder
Using Metasploit against systems you do not own or have explicit written permission to test is illegal. Only use it in authorised lab environments, CTFs, or on your own infrastructure.
02

Installation

Choose your platform. Both paths end with a working msfconsole — the commands are different but the outcome is the same.

On Kali, Metasploit is available directly from the official repos. Note that on a fresh Kali install it is usually pre-installed — check first before re-installing.

bash 
$ sudo apt update
$ sudo apt install metasploit-framework -y
Pre-installed on Kali?
Run msfconsole --version first. If it returns a version number, Metasploit is already installed and you just need to update it with sudo apt update && sudo apt upgrade metasploit-framework.

apt pulls in Ruby, PostgreSQL, and all other dependencies automatically. The download is around 500 MB — it may take a few minutes.

On Termux (inside NetHunter or standalone), Metasploit lives in the unstable repo. Add it first, then install.

bash — Step 1: Update Termux 
$ pkg update && pkg upgrade
bash — Step 2: Add unstable repo 
$ pkg install unstable-repo
bash — Step 3: Install Metasploit 
$ pkg install metasploit
Why unstable-repo?
Metasploit is not in Termux's main package repo — it's hosted in the unstable repo due to its size and frequent updates. This is normal and expected, not a warning about stability of your system.
pkg install metasploit running in Termux — this is a large download, keep your screen awake and stay on a solid Wi-Fi connection
03

Verify & Launch

Once installed, check the version first, then launch the console. These commands are the same on both Kali and Termux.

Check the version:

bash 
$ msfconsole --version
Framework Version: 6.4.135-dev

Launch msfconsole:

bash 
$ msfconsole
_ _ / \ / \ | | _ _ _ _ _ _ | | | | \/\/ /\ / \ / \ / \ / \ / \ / \ /\ | | | | /\ / \| | | | | | | | |/ \| | | | / \ /----+ | | | | | | | +----+ | \_\/ \ / / \_ | _/ \_/ \_/ \_/ \_/ \ \/ \/ \/ =[ metasploit v6.4.135-dev ] + -- --=[ 2430 exploits - 1255 auxiliary - 413 post ] + -- --=[ 1474 payloads - 47 encoders - 11 nops ] + -- --=[ 9 evasion ] msf6 >

On Kali, you may be asked on first launch if you want to set up a PostgreSQL database — type y. A connected database makes module searches significantly faster. On Termux the database is not set up automatically; the console still works fully without it.

04

Essential msfconsole Commands

Once inside the msf6 > prompt, these are the core commands you'll use constantly.

Core msfconsole commands
CommandWhat it does
helpShow all available console commands
search <term>Search for exploits, modules, CVEs by keyword
use <module>Load a module (e.g. use exploit/multi/handler)
show optionsList required and optional settings for a loaded module
set RHOSTS <ip>Set the target IP address
set LHOST <ip>Set your local IP (for reverse shells)
run / exploitExecute the loaded module
db_statusCheck if PostgreSQL database is connected
exitExit msfconsole

Example — searching for EternalBlue:

msfconsole 
msf6 > search eternalblue

05

Troubleshooting

msfconsole: command not found on Kali
Run sudo apt install metasploit-framework -y. If the package doesn't exist, your sources may be outdated — run sudo apt update first and try again.
msfconsole: command not found on Termux
Make sure you installed unstable-repo first — pkg install unstable-repo — then re-run pkg install metasploit. Without the unstable repo, Termux can't find the package.
db_status shows not connected (Kali)
PostgreSQL isn't running. Start it and initialise the database with sudo msfdb init then relaunch msfconsole. A connected database speeds up module searches significantly.
msfconsole is extremely slow to load on Termux
This is normal on first launch — Ruby initialising on Android is slow. Subsequent launches are faster. Keep your device plugged in and avoid running other heavy apps simultaneously.
Antivirus blocking install on a Windows dual-boot / shared folder
Antivirus engines flag Metasploit as malware because it contains real exploits. Disable real-time protection on Windows before installing, or exclude the Metasploit directory from scanning. This does not affect a pure Kali/Termux install.
