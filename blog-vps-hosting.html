Self-Hosting Guide

VPS & Self-Hosted VPN

Why self-hosting your VPN is the gold standard for privacy — and how to do it step by step on DigitalOcean or Vultr. From spinning up a droplet to connecting your first client in under an hour.

💰 High-value skill ⏱️ ~1 hour setup 🏷️ Intermediate

Why Self-Host Your VPN?

Commercial VPN providers have improved dramatically in recent years, and many offer legitimate privacy protections. But there are compelling reasons why anyone serious about network privacy should consider running their own VPN server on a Virtual Private Server (VPS).

Complete control over your data. When you use a commercial VPN, you're trusting the provider's no-logs policy. But you have no way to verify it. They could be logging your traffic and handing it to authorities under legal compulsion — and you'd never know. When you run your own server, you control what gets logged (ideally, nothing).

A fresh, unlisted IP address. Commercial VPN providers' IP ranges are well-known and actively blacklisted by streaming services, websites, and carrier DPI systems. A freshly provisioned VPS has an IP address that isn't pre-categorized as VPN traffic — significantly improving both access and reliability.

Full configuration control. You choose the protocol, the cipher suite, the port, the DNS server, the routing rules. You can configure your server exactly as needed for your specific use case — something commercial providers with generic configurations can't offer.

Cost efficiency at scale. For a single user, a commercial VPN may be cheaper. For a family or small team sharing a server, a $5/month VPS beats most commercial VPN pricing while offering better privacy.

What Is a VPS?

A Virtual Private Server (VPS) is a virtual machine that runs on shared physical hardware in a data center. Unlike shared hosting (where many customers share the same software environment), a VPS gives you your own isolated OS instance with dedicated resources — a specified amount of CPU, RAM, and storage that's guaranteed to be available to you.

From your perspective, a VPS looks and behaves exactly like a dedicated server. You get root/admin access. You can install any software. You configure it however you like. The "virtual" part just means it's running as a VM on physical hardware that also hosts other VPS instances — but those instances are isolated from yours.

For a VPN server, you need very modest resources. A $4–6/month VPS with 512MB–1GB RAM and 1 shared vCPU is perfectly adequate for a personal or small-family VPN server running OpenVPN or WireGuard.

Choosing a VPS Provider

DigitalOcean

DigitalOcean is one of the most developer-friendly VPS providers available. Its product is called "Droplets" — clean, simple virtual machines that can be provisioned in seconds. DigitalOcean has an excellent UI, good documentation, and data centers in New York, San Francisco, Amsterdam, Singapore, London, Frankfurt, Toronto, Bangalore, and Sydney — giving you good options for choosing a server location that minimizes latency or maximizes geographic utility.

Pricing starts at $4/month for a basic Droplet (512MB RAM, 1 vCPU, 10GB SSD, 500GB transfer). For most personal VPN use, this is entirely sufficient.

DigitalOcean also offers a $200 free credit for new accounts — meaning you can run a VPN server for free for several months while you learn the system.

Vultr

Vultr is a strong competitor to DigitalOcean with slightly lower base pricing and an extensive global network of data centers — 32 locations as of 2026, including locations in Africa (Johannesburg) and South America that DigitalOcean doesn't cover. For users in regions where latency to DigitalOcean's nearest datacenter is high, Vultr's geographic coverage can make a significant difference.

Vultr's "Cloud Compute" instances start at $2.50/month (512MB RAM, 1 vCPU, 10GB SSD, 500GB bandwidth). Their interface is clean and the API is well-documented.

Setting Up Your VPS Server

This guide uses Ubuntu 22.04 LTS — the most widely used Linux distribution for VPS deployments and the one with the most comprehensive documentation and community support.

Step 1: Create Your Server

Log in to DigitalOcean or Vultr and create a new server instance:

  • OS: Ubuntu 22.04 LTS
  • Size: Smallest available ($4–6/month) is sufficient
  • Region: Choose the closest to you for lowest latency, or a specific country for geographic reasons
  • Authentication: SSH key (more secure) or password. If using password, make it very strong.

Step 2: Connect to Your Server

# Connect via SSH (replace with your server's IP)
ssh root@YOUR_SERVER_IP

# You'll see something like:
Welcome to Ubuntu 22.04.3 LTS
root@your-server:~#

Step 3: Update the System

# Always update first
apt update && apt upgrade -y

# Install essential tools
apt install -y ufw curl wget nano fail2ban

Step 4: Basic Security Hardening

# Configure firewall
ufw allow OpenSSH
ufw allow 443/tcp    # VPN port
ufw allow 1194/udp   # OpenVPN default UDP port
ufw enable

# Disable root SSH password login (use keys only)
sed -i 's/PermitRootLogin yes/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
systemctl restart ssh

Installing OpenVPN

The easiest way to install a production-ready OpenVPN server is using the angristan OpenVPN install script — a widely trusted, open-source automated installer that handles certificate generation, configuration, and service setup.

# Download the installer
curl -O https://raw.githubusercontent.com/angristan/openvpn-install/master/openvpn-install.sh

# Make it executable
chmod +x openvpn-install.sh

# Run the installer
bash openvpn-install.sh

The script will ask you several questions:

  • IP address: It auto-detects your server's public IP. Confirm it.
  • Port: Default is 1194 UDP. Change to 443 TCP if you need firewall bypass.
  • DNS: Choose a privacy-respecting DNS server (Cloudflare 1.1.1.1 or AdGuard)
  • Client name: Name for your first client configuration (e.g., "myphone")

After completion, the script generates a .ovpn file in your home directory. This is the configuration file you transfer to your phone or computer to connect.

# Transfer the .ovpn file to your local machine
# Run this on your LOCAL machine (not the server):
scp root@YOUR_SERVER_IP:~/client.ovpn ./client.ovpn

# Or for Android, use any SFTP app to download it
# Then import into OpenVPN for Android

Installing WireGuard (Faster Alternative)

For users who don't need the firewall-bypass capabilities of OpenVPN-over-TCP and just want maximum speed, WireGuard is the better choice:

# WireGuard installer (also by angristan)
curl -O https://raw.githubusercontent.com/angristan/wireguard-install/master/wireguard-install.sh
chmod +x wireguard-install.sh
bash wireguard-install.sh

WireGuard generates a QR code at the end of installation — scan it with the WireGuard Android app to configure your phone instantly.

Setting Up an SSH Server for HA Tunnel / HTTP Injector

If you want to use your VPS as the SSH server for HA Tunnel Plus or HTTP Injector configs, you don't need to install additional software — Ubuntu has OpenSSH Server included. You just need to configure it correctly.

# Check SSH is running
systemctl status ssh

# Create a dedicated VPN user (don't use root for tunnel clients)
adduser vpnuser
passwd vpnuser

# Optionally allow SSH on port 443 as well as 22
echo "Port 22" >> /etc/ssh/sshd_config
echo "Port 443" >> /etc/ssh/sshd_config
systemctl restart ssh
ufw allow 22/tcp
ufw allow 443/tcp

Your SSH server details for HTTP Injector / HA Tunnel Plus configs will be:

  • Server: Your VPS IP address
  • Port: 22 or 443 (whichever you've opened)
  • Username: vpnuser (or root, though this is less recommended)
  • Password: The password you set
✅

You're Done

You now have a self-hosted VPN server. Use the Setup Guides to connect your Android device, or import your .ovpn file directly into OpenVPN for Android. Welcome to full network ownership.

Monitoring Your Server

Once your server is running, basic monitoring is important to know it's healthy and hasn't been compromised.

# Check connected OpenVPN clients
cat /var/log/openvpn/status.log

# Check server resource usage
htop

# Check for failed SSH login attempts (intrusion detection)
fail2ban-client status sshd

# Check bandwidth usage (install vnstat first)
apt install vnstat
vnstat -d   # Daily stats
vnstat -m   # Monthly stats
