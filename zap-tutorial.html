Installing ZAP — Web App Scanner Setup Guide

Install the Zed Attack Proxy (ZAP by Checkmarx) on Kali Linux, launch the GUI, and run your first automated web vulnerability scan in minutes.

Your progress 0%
01

What is ZAP?

ZAP intercepts web traffic and actively probes applications for vulnerabilities — the GUI makes it accessible to beginners and experts alike

ZAP (Zed Attack Proxy) by Checkmarx is the world's most widely used open-source web application scanner. Originally an OWASP project, it helps developers and penetration testers automatically find security vulnerabilities — SQL injection, XSS, CSRF, and all OWASP Top 10 issues — in web applications.

⚠ Desktop environment required
ZAP is a GUI-only application — it requires a running desktop environment to launch. On a standard PC or laptop running Kali this works out of the box. On Android / NetHunter you must have a desktop session running first via KEX (kex &), Termux-X11, or TigerVNC before starting ZAP.
What you can do with ZAP
Run automated vulnerability scans, intercept and modify HTTP/HTTPS traffic, spider web applications to discover hidden endpoints, replay and fuzz requests, and generate detailed security reports — all from a single GUI.
02

Installing ZAP

ZAP is packaged in Kali's repositories as zaproxy. A single apt command is all you need — it pulls in Java and all other dependencies automatically.

bash 
$ sudo apt update
$ sudo apt install zaproxy -y
bash 
$ sudo apt update
$ sudo apt install zaproxy -y
Not in standard Ubuntu repos
On plain Ubuntu, zaproxy may not be available. If the install fails, use the official installer from zaproxy.org/download or add Kali's tool repos first.

The download includes Java runtime, ZAP core, and all bundled add-ons. It may take a minute or two to complete.

03

Starting ZAP

With installation complete, launch ZAP from a terminal inside your desktop session. Remember — a display must be active before running this command.

bash 
$ zaproxy
OWASP ZAP 2.16.0 starting …
[main] INFO  org.zaproxy.zap.ZAP - OWASP ZAP 2.16.0
[main] INFO  org.zaproxy.zap.ZAP - Platform: Linux amd64
[main] INFO  org.zaproxy.zap.ZAP - Java: Eclipse Adoptium 17.0.9
[main] INFO  org.zaproxy.zap.ZAP - Locale: en_GB
[ZAP] INFO  org.zaproxy.zap.control.AddOnLoader - Loading add-ons from …

ZAP will take 10–20 seconds to load on first launch while it initialises its database and add-ons. On subsequent launches it's faster. The GUI will open automatically once ready.

First-launch session prompt
On first launch ZAP asks whether to persist the session. For learning purposes choose "No, I do not want to persist this session" — you can enable sessions later when you need to save your work.
ZAP's GUI open and ready — the main window shows the Sites tree on the left, the request/response panel on the right, and alerts at the bottom
04

Basic Use — Your First Scan

Once ZAP is open, running your first scan takes three clicks. Always test against targets you own or have explicit permission to scan. The URL below is a legal practice target provided by Acunetix specifically for this purpose.

Legal reminder
Scanning web applications you do not own or have written permission to test is illegal. Use intentionally vulnerable practice apps like testphp.vulnweb.com, DVWA, or WebGoat for learning.
1
Choose a scan mode
On the ZAP welcome screen select Automated Scan for a hands-off full scan, or Manual Explore to browse the target yourself while ZAP passively records traffic.
2
Enter the target URL
Type the target into the URL field. For a safe practice run use: http://testphp.vulnweb.com
3
Click Start
ZAP will spider the site, then run active scans. Alerts appear in the bottom panel colour-coded by severity — Red = High, Orange = Medium, Yellow = Low, Blue = Informational.
ZAP alert severity levels
ColourSeverityMeaning
RedHighCritical vulnerabilities — SQL injection, RCE, auth bypass
OrangeMediumXSS, CSRF, sensitive data exposure
YellowLowMissing security headers, cookie flags
BlueInformationalNotes on tech stack, server version disclosure
GreenFalse PositiveMarked by you as not a real issue
05

Troubleshooting

Common issues when getting ZAP running for the first time.

zaproxy: command not found
The package isn't installed or the path isn't set. Run sudo apt install zaproxy -y again. If it still fails, the package may not be in your current repos — check with apt-cache search zaproxy.
cannot open display or GUI won't appear
No desktop is running. On NetHunter start KEX first with kex &, or launch Termux-X11 or TigerVNC, then run zaproxy from a terminal inside that session.
ZAP loads but scan finds nothing
The spider may have been blocked by the target or timed out. Make sure the target URL is reachable (curl http://testphp.vulnweb.com), then try increasing the spider timeout under Tools → Options → Spider.
Java version error on launch
ZAP now requires Java 17 or newer. Check your version with java -version. Install the correct version with sudo apt install openjdk-17-jre -y then relaunch.
